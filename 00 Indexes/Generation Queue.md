# 03 Reading / Generation Queue

## Purpose

This note serves as the operational tracking center for the *History of Big Ideas* vault. It records what has been generated, what remains to be written, and how the work is grouped into manageable batches. Unlike interpretive notes, this page is not primarily analytical. Its function is organizational: it keeps the project coherent, prevents duplication, and makes the overall construction of the vault visible at a glance.

It is also the practical bridge between the canon itself and the generation workflow. The vault is designed as a structured intellectual system with fixed inventories for books, concepts, authors, eras, and structural navigation notes. Because of that, progress must be tracked against canonical filenames and official note counts, not informal memory or improvised labels.

## Workflow Status Key

- **Not Started** — no draft has yet been generated.
- **In Progress** — currently being worked on or actively being generated.
- **Draft Generated** — a full note has been produced, but may still need revision or consistency review.
- **Revised** — the note has been meaningfully improved for structure, clarity, or linking.
- **Finalized** — the note is complete, canonically structured, and ready to stand as part of the stable vault.

## Notes to Self

- Keep all filenames strictly canonical across every note, map, and index.
- Preserve three-digit numbering for all book notes without exception.
- Use structural notes as navigation tools rather than interpretive essays.
- Treat concepts, eras, authors, and books as parts of one integrated system rather than separate mini-projects.
- Move notes from **Draft Generated** to **Revised** only after checking structure, clarity, and link consistency.
- Move notes to **Finalized** only after confirming that headings, filenames, and wikilinks match the canonical source exactly.
- Prefer steady batch completion over irregular generation, since sequence supports consistency.
- Check author-name normalization carefully, especially where shortened or familiar forms are common.
- Keep the project focused on generation, revision, and integration rather than redesign.
- Use this page as the single operational overview for vault completion.
